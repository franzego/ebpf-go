# Building a Simple Kernel-Level Network Monitor/visualiser with eBPF and Go

A ground-up journey from zero eBPF knowledge to a working kernel-level TCP connection visualiser.

---

## What This Is

A real-time network visualiser/visualiser that hooks directly into the Linux kernel and captures every outbound TCP connection — the process name, source IP, destination IP, source port, and destination port — before the packet even leaves the kernel. This is a simple exposition on ebpf and go. 

I came into making this project with zero knowledge on what ebpf was. Having read the documentation - which was super useful and explanatory and using claude in teaching mode, I started to get the concept. Its a simple one at that. **Instead of rewriting or editing the linux kernel module - which can lead to unexpected behaviour including the crashing of the kernel, we hook EBPF programs into the kernel.** 

The OS is divided into the Userspace - where our Go, Rust or higher level languages run - and the Kernel space. The kernel space is shut out from the userspace and can only be accessed by request thorough syscalls APIs. Further explanations are below.


Built with:
- **eBPF C** — the kernel-side hook
- **Go + cilium/ebpf** — the userspace brain
- **bpf2go** — the code generation glue

---

## How It Works

```
the visualiser.c (C code)
      ↓  clang compiles it
bytecode inside visualiser.o
      ↓  bpf2go embeds it into Go binary
go run main.go
      ↓  loads bytecode into kernel (once)
kernel: "i now have this eBPF program,
         i'll run it every time tcp_connect fires"
      ↓
tcp_connect fires → eBPF captures event → writes to ring buffer
      ↓
Go reads from ring buffer → formats and prints
```

## Setup

**1. Generate vmlinux.h from your running kernel**
```bash
bpftool btf dump file /sys/kernel/btf/vmlinux format c > vmlinux.h
```
This is always the first step. `vmlinux.h` contains every kernel type definition your C code needs.

**2. Write your eBPF C code**

See `ebpf/visualiser.c`. Key parts:
- `SEC("fentry/tcp_connect")` — hooks directly onto the kernel's `tcp_connect` function
- `BPF_MAP_TYPE_RINGBUF` — shared memory buffer between kernel and userspace
- `bpf_ringbuf_reserve` + `bpf_ringbuf_submit` — write events to the buffer

**3. Add the go:generate directive**
```go
//go:generate go run github.com/cilium/ebpf/cmd/bpf2go visualiser ebpf/visualiser.c
```

**4. Run code generation**
```bash
go generate
```
This compiles `visualiser.c`, embeds the bytecode, and generates typed Go bindings.

**5. Install dependencies**
```bash
go mod tidy
```

**6. Build**
```bash
go build -o visualiser .
```

**7. Run (requires root)**
```bash
sudo ./visualiser
```

## The Kernel-Side Code (ebpf/visualiser.c)

```c
// go:build ignore
#include "vmlinux.h"
#include <bpf/bpf_endian.h>
#include <bpf/bpf_helpers.h>
#include <bpf/bpf_tracing.h>

// Every eBPF program must declare a GPL-compatible license
// to unlock the kernel's advanced helper functions.
char __license[] SEC("license") = "GPL";

// 1. The structural blueprint of the data we want to collect.
// This exact layout will be read by our Go application.
struct connection_event {
  u32 src_ip;
  u32 dst_ip;
  u16 src_port;
  u16 dst_port;
  u8 comm[16]; // Stores the 16-character process name (e.g., "curl", "nginx")
};

// 2. Define the lockless shared memory conveyor belt (Ring Buffer Map).
struct {
  __uint(type, BPF_MAP_TYPE_RINGBUF);
  __uint(max_entries, 1 << 16); // Allocation size: 64KB
} conn_events SEC(".maps");

// 3. The Camera Hook: Tell the kernel to run this code
// the exact microsecond any process enters 'tcp_connect'.
SEC("fentry/tcp_connect")
int BPF_PROG(tcp_connect, struct sock *sk) {

  // Extract the socket's address family (IPv4 vs IPv6).
  // skc_family lives deep inside the kernel's socket tracking structures.
  u16 family = 0;
  if (bpf_probe_read_kernel(&family, sizeof(family),
                            &sk->__sk_common.skc_family) < 0) {
    return 0;
  }

  // Filter: If it's not an IPv4 connection (AF_INET is 2), ignore it.
  // This keeps our personal project simple and clean.
  if (family != 2) {
    return 0;
  }

  // Safely reserve a slot on our Ring Buffer memory conveyor belt.
  struct connection_event *ev;
  ev = bpf_ringbuf_reserve(&conn_events, sizeof(*ev), 0);
  if (!ev) {
    return 0; // Buffer is full; drop this telemetry event to protect kernel
              // performance
  }

  // Read the binary IP data directly from the kernel socket pointers
  if (bpf_probe_read_kernel(&ev->src_ip, sizeof(ev->src_ip),
                            &sk->__sk_common.skc_rcv_saddr) < 0) {
    bpf_ringbuf_discard(ev, 0);
    return 0;
  }
  if (bpf_probe_read_kernel(&ev->dst_ip, sizeof(ev->dst_ip),
                            &sk->__sk_common.skc_daddr) < 0) {
    bpf_ringbuf_discard(ev, 0);
    return 0;
  }

  // Read the ports. The kernel stores the destination port in network byte
  // order (Big Endian). We convert it to host byte order (Little Endian on
  // standard x86/ARM hardware) using bpf_ntohs.
  u16 dport = 0;
  if (bpf_probe_read_kernel(&dport, sizeof(dport),
                            &sk->__sk_common.skc_dport) < 0) {
    bpf_ringbuf_discard(ev, 0);
    return 0;
  }
  ev->dst_port = bpf_ntohs(dport);

  if (bpf_probe_read_kernel(&ev->src_port, sizeof(ev->src_port),
                            &sk->__sk_common.skc_num) < 0) {
    bpf_ringbuf_discard(ev, 0);
    return 0;
  }

  // Ask the kernel context to copy the name of the executable
  // occupying the CPU into our event structure.
  if (bpf_get_current_comm(&ev->comm, sizeof(ev->comm)) < 0) {
    bpf_ringbuf_discard(ev, 0);
    return 0;
  }

  // Submit the data slot. This instantly notifies our user-space Go program.
  bpf_ringbuf_submit(ev, 0);

  return 0;
}
```

## The Userspace Code (main.go)

```go
package main

//go:generate go run github.com/cilium/ebpf/cmd/bpf2go visualiser ebpf/visualiser.c

// Go struct must mirror the C struct exactly
// same fields, same order, same sizes
type ConnectionEvent struct {
    SrcIP   uint32
    DstIP   uint32
    SrcPort uint16
    DstPort uint16
    Comm    [16]byte
}

func main() {
    // 1. load generated objects into kernel
    objs := visualiserObjects{}
    loadVisualiserObjects(&objs, nil)
    defer objs.Close()

    // 2. attach to fentry hook — program now fires on every tcp_connect
    lnk, _ := link.AttachTracing(link.TracingOptions{
        Program: objs.TcpConnect,
    })
    defer lnk.Close()

    // 3. open ring buffer reader
    rd, _ := ringbuf.NewReader(objs.ConnEvents)
    defer rd.Close()

    // 4. event loop — blocks, reads every event the kernel sends up
    for {
        record, _ := rd.Read()
        var event ConnectionEvent
        binary.Read(bytes.NewReader(record.RawSample), binary.LittleEndian, &event)
        // format and print...
    }
}
```

## Key Concepts Learned

**eBPF**
- Runs sandboxed programs inside the kernel without changing kernel source
- Event-driven — attaches to hooks and fires when events occur
- Programs are written in C, compiled to bytecode via clang/LLVM
- The kernel verifier checks every program before loading

**Hook Types**
| Hook | Usage |
|---|---|
| `fentry` | Hook kernel functions directly, typed args, modern |
| `kprobe` | Hook kernel functions, older approach via registers |
| `tp/syscalls/...` | Stable tracepoints at syscall boundary, portable |

**Maps**
- Shared memory between kernel and userspace
- Ring buffer (`BPF_MAP_TYPE_RINGBUF`) is the modern choice — single buffer, all CPU cores have access to it, efficient
- Perf event array is the older approach — one buffer per CPU

**vmlinux.h**
- Generated from the running kernel: `bpftool btf dump file /sys/kernel/btf/vmlinux format c > vmlinux.h`
- Provides all kernel type definitions your C code needs
- Must be generated first before writing any C code

**bpf2go**
- Compiles your C code automatically
- Embeds the `.o` bytecode directly into your Go binary
- Generates typed Go bindings — no more string-based map/program references which could be error prone
- Run after writing C, before writing Go

**Kernel space vs Userspace**
```
KERNEL SPACE                    USER SPACE
────────────────────────────────────────────
C eBPF program lives here       Go program lives here
captures events                 processes events
writes to ring buffer           reads from ring buffer
no complex logic                all your business logic
```
